YAML schema for battery keyword.

The battery keyword adds a Battery Component to the Network.

- battery:
id: <string> # Component ID.
inverter_id: <string> # Inverter to which battery is connected.
dt: <hh:mm:ss> # Component will update every dt.
init_soc: <real> # Initial state of charge, MWh.
max_soc: <real> # Maximum state of charge, MWh.
max_charge_power: <real> # Maximum charging power, MW.
max_discharge_power: <real> # Maximum discharge power, MW.
charge_efficiency: <real> # Charging efficiency; d/dt SOC = P_chg * eff.
discharge_efficiency: <real> # Discharge efficiency; d/dt SOC = -P_dis / eff.
requested_power: <real> # Requested output DC power for battery, MW.

YAML schema for heartbeat keyword.

The heartbeat keyword adds a Heartbeat Component to the Network.

- heartbeat:
id: <string> # Component ID.
dt: <hh:mm:ss> # Component will update every dt.
slaves: [<string>, <string>, ...] # Optional Components to update when I update.

YAML schema for inverter keyword.

The inverter keyword adds an Inverter Component to the Network.

- inverter:
id: <string> # Component ID.
sim_network_id: <string> # SimNetwork to which inverter belongs.
bus_id: <string> # Bus to which inverter is attached.
phases: <phases> # Bus Phases, e.g. [A, B, C] to which power is produced.
is_delta: <bool> # If true, use a Delta connection.
efficiency_dc_to_ac: <real> # Efficiency when injecting grid power.
efficiency_ac_to_dc: <real> # Efficiency when consuming grid power.
max_S_mag: <real> # Max apparent power |S|, MVA.
requested_Q: <real> # Requested reactive power, MVAr. May curtail for max_S_mag.

- simulation:

YAML schema for matpower keyword.

The matpower keyword reads a matpower (.m) file and creates a SimNetwork.

- matpower:
sim_network_id: <string> # ID of the SimNetwork Component into which to parser matpower data.
# Other parameters as per the matpower keyword for network parsers (see below).

Please see YAML specification for Networks (matpower keyword) for additional keywords.

YAML schema for sim_network keyword.

The sim_network keyword adds a new SimNetwork Component to the Simulation.

- sim_network:
id: <string> # Component ID.
P_base: <real> # Base power, MW.
freq_Hz: <real> # Nominal frequency in Hz.
solver: <string> # Assigned network solver: currently "nr" (Newton-Raphson) or "opf_pt" (PowerTools OPF).
network: <string> # Complete network specification, see below.

Please see YAML specification for Networks for documentation about how to fill in the "network" keyword above.

YAML schema for simple_building keyword.

The simple_building keyword adds a new SimpleBuilding Component to the Simulation.

- simple_building:
id: <string> # Component ID.
bus_id: <string> # ID of bus to which building is attached.
zip_id: <string> # ID of network ZIP corresponding to the building. Defaults to id.
sim_network_id: <string> # ID of the SimNetwork.
dt: <hh:mm:ss> # Update frequency.
kb: <real> # Lumped thermal conductivity, W/K.
cb: <real> # Heat capacity of building, J/K
Tb_init: <real> # Initial temperature of building.
kh: <real> # HVAC PID parameter, W/K.
COP_cool: <real> # Coefficient of performance, cooling.
COP_heat: <real> # Coefficient of performance, heating.
P_max: <real> # HVAC maximum power, W.
Ts: <real> # Temperature setpoint, 째C.
internal_heat_power: <string> # ID of TimeSeries -> heat (W) gen. by occupants, appliances etc.
weather: <string> # ID of weather component.
